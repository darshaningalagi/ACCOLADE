Coding Event Rules - ACCOLADE #11.25

1. Eligibility

  • Participation is open only to 1st, 3rd, and 5th semester students.
  • Each team must have 1 participant from each eligible semester (one from 1st, one from 3rd, and one from 5th).
  • Cross-department teams are allowed/not allowed based on event rules.

2. Team Formation

  • Each team should have exactly 3 members.
  • Team name must be submitted at registration.
  • Once registered, no changes in team members will be allowed.

3. Event Rounds

  • MCQ Round – Aptitude + Programming basics.
  • Coding Round – Solve problems within a time limit.
  • Debugging Round – Fix errors in a given code.
  • Final Round (Project/Problem Statement) – Build logic or small application (optional).

4. Rules During the Event

  • Use of mobile phones, internet browsing, or external resources is strictly prohibited (unless specified by organizers).
  • Participants must use the system and compiler provided by organizers.
  • Code plagiarism (copy-paste from others/online) will lead to disqualification.
  • Each round will have a time limit; late submissions will not be accepted.
