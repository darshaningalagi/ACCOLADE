ACCOLADE #11.25 - Coding Event Rounds & Rules

Round 1: Dwarka Darshan

  • Each team must consist of 3 members.
  • Teams have to find hidden QR codes placed at different spots in the college.
  • Once the QR code is found, submit it to the volunteers present at the initial spot.
  • Scoring: Correct answer = 5 points awarded; Wrong answer = No points.
  • Time Limit: 45 minutes for teams to complete the round.
  • No team is allowed to damage QR codes.
  • Using external help (like searching online) may lead to disqualification.
  • Teams with the highest points & fastest completion qualify for the next round.

Round 2: Arjun’s Aim

  • Each team must consist of 3 members.
  • Role Division:
    • Participant 1 → Coding (write the solution for the given problem).
    • Participant 2 → Debugging (fix errors in a buggy code).
    • Participant 3 → Code Optimization (improve existing code for efficiency).
  • All three participants start their respective tasks at the same time.
  • Time Limit: 90 minutes for completion.
  • Scoring System:
    • Coding: Correctness & logic.
    • Debugging: Number of errors fixed.
    • Optimization: Efficiency, readability, and reduced complexity.
  • No role exchange after the round begins, and no discussion/code-sharing during the tasks.
  • Any use of unfair means or outside help will lead to disqualification.
  • Team with the highest combined score qualifies for the next round.

Round 3: Yudh of Yudhisthira

  • Each team will have 3 participants.
  • Quiz Details:
    • Topics: Java, Python, C, and General Programming Concepts.
    • Total Duration: 30 minutes.
    • Quiz comprises MCQs, output-based, and conceptual coding logic questions.
  • Scoring: Each correct answer earns points. No negative marking.
  • Final score is the team’s total.
  • No external help allowed (Google, ChatGPT, etc.); discussion is permitted only within the team.
  • Any unfair practice leads to disqualification.
  • Team with the highest quiz score within 30 minutes wins this round.

Round 4: Chakravyuh

  • Each team will have 3 members.
  • Teams must write code to generate a given pattern (problem given at the start).
  • All members work together on the same code; teamwork is essential.
  • Time Limit: 60 minutes to complete the pattern.
  • Scoring:
    • Correct Output: Matches the required pattern.
    • Code Quality: Indentation, readability, and logic.
    • Efficiency: Minimal and optimized code.
  • Only the provided system/editor may be used. No use of unauthorized resources.
  • Team that generates the correct pattern fastest and with clean, efficient code wins.

Round 5: Krishnas Council

  • A problem statement is given to each team.
  • Teams must provide a solution and create a prototype based on their solution.
  • Time Limit: 90 minutes to develop the prototype.
  • Scoring Criteria:
    • Innovation: Creativity and originality of the solution.
    • Functionality: How well the prototype works.
    • Presentation: Clarity and effectiveness of the presentation.
  • Each team must have their own devices for prototype development.
  • Teams can use any programming language or tools they are comfortable with.
  • The team with the most innovative, functional, and well-presented prototype wins.
